# How to Use the Streamlit App

> **Audience:** Team members and stakeholders who want to interact with the project through the GUI.
>
> **Goal:** Use all 7 pages of the Streamlit application for data management, training, prediction, and results analysis.

---

## Launching the App

```bash
streamlit run app/Home.py
```

The app opens at `http://localhost:8501`. Navigation is via the sidebar.

### Dataset Manager (Page 2)

**Purpose:** Generate or upload data, preview statistics, preprocess, and train models.

**4 Sections:**

1. **Get Dataset**
   - Click **Generate Synthetic Dataset** for instant data
   - Or upload a CSV/Excel file
   - Configure row count (10k–200k) and user count (500–10k)

2. **Preview & Statistics**
   - View raw dataframe with scrolling
   - Metric cards: total rows, users, genres, date range
   - Per-genre watch-time distribution

3. **Preprocess**
   - Click **Run Preprocessing Pipeline**
   - Progress bar shows: Clean → IQR Outlier Removal → One-Hot Encode → Scale
   - Summary: rows removed, outliers removed, final shape

4. **Train Models**
   - Select task: `Both`, `CBCB-S Only`, `CBCB-R Only`
   - Toggle: **Hyperparameter Tuning** (GridSearchCV — slower but better)
   - Toggle: **Include Boosters** (requires xgboost/lightgbm/catboost)
   - Click **Train Selected Models**
   - Progress bars show per-model training status

### CBCB Architecture (Page 3)

**Purpose:** Visual explanation of the framework — no actions required.

- **End-to-End Pipeline** — Graphviz flowchart of the full pipeline
- **CBCB-S Labelling Rule** — with example sequence and labels
- **CBCB-R Labelling Rule** — with example sequence and ternary labels
- **Data-Flow Diagram** — how data transforms at each stage
- **Preprocessing Equation:** `X' = fs(fe(fc(X)))`

### Prediction (Page 4)

**Purpose:** Live inference on user behaviour.

**Input Form:**

| Field | Type | Example |
|---|---|---|
| User ID | text | `U_42` |
| Current Genre | dropdown | `Action` |
| Program Class | dropdown | `Movie` |
| Watch Duration | slider (0–100%) | 75% |
| Previous Genres | multiselect | `["Drama", "Comedy"]` |
| Model | dropdown | `Random Forest` |

**Output:**

- CBCB-S Prediction — "Repeat" or "No Repeat" with confidence
- CBCB-R Prediction — "Repeat", "Revert", or "No Repeat" with confidence
- Predicted Next Genre — heuristic from user history
- Engagement Gauge — watch-duration percentile
- Confidence Gauge — model's prediction probability

**How inference works:**

```python
# app/pages/3_Prediction.py — simplified
def _predict(task):
    row = _build_single_row_features(task)
    model = load_trained_model(task, selected_model)
    scaler = model['scaler']
    X_scaled = scaler.transform(row[X_cols])
    pred = model['model'].predict(X_scaled)[0]
    proba = model['model'].predict_proba(X_scaled)[0]
    return pred, max(proba)
```

### Experimental Results (Page 5)

**Purpose:** Detailed model performance visualisation.

- **Performance Table** — color-coded dataframe (green = best)
- **Metric Bar Charts** — 2×2 grid (Accuracy, Precision, Recall, F1)
- **Confusion Matrices** — one heatmap per model per task
- **Feature Importance** — horizontal bar chart per model
- **Best Model + Recommendation** — highlighted best per metric

### Model Comparison (Page 6)

**Purpose:** Cross-task model comparison.

- **Radar Chart** — Plotly interactive, one trace per model across 4 metrics
- **Grouped Bar Chart** — Plotly, side-by-side bars per model per task
- **Leaderboard** — combined CBCB-S + CBCB-R mean score, sorted descending

The leaderboard is generated by:

```python
# app/pages/5_Model_Comparison.py — simplified
leaderboard = []
for model in model_names:
    s_metrics = metrics['cbcb_s'][model]
    r_metrics = metrics['cbcb_r'][model]
    mean_f1 = (s_metrics['f1'] + r_metrics['f1']) / 2
    leaderboard.append({'Model': model, 'CBCB-S F1': s_metrics['f1'],
                        'CBCB-R F1': r_metrics['f1'], 'Mean': mean_f1})
leaderboard.sort(key=lambda x: x['Mean'], reverse=True)
```

### About (Page 7)

## Tips

- Models must be trained first (via CLI or Page 2) before Pages 4–6 work.
- Use `st.session_state` is only for `raw_df` and `processed_df` within a session.
- Refresh the app after training new models to reload `metrics.json`.
- All pages import from `_shared.py` which injects custom CSS and sets up Python paths.
